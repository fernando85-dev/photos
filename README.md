# Photo Album Management System

## Description

This application allows users to manage digital photo albums. Key features include creating albums, uploading photos with notes to these albums, and customizing the application's appearance with different color themes. It also includes a (simulated) feature for sharing albums by emailing a zip file of selected photos.

The application is built using Python and the Flask web framework, with an SQLite database for data storage. It is designed to be containerized using Docker for ease of deployment.

## Features

*   **Album Management:** Create named photo albums.
*   **Photo Uploads:** Upload photos (jpg, jpeg, png, gif) to albums.
*   **Photo Notes:** Add textual notes to each uploaded photo.
*   **Theme Customization:** Switch between different color themes (e.g., white, black, blue, green) for the user interface.
*   **File Serving:** Serves uploaded photos securely.
*   **(Simulated) Email Sharing:** Package photos from an album into a zip file and simulate sending them via email.
*   **Database Storage:** Uses SQLite to store album and photo metadata.
*   **Dockerized:** Includes a `Dockerfile` for building a container image and `.dockerignore` for optimized builds.
*   **Unit Tested:** Core application logic is covered by unit tests.

## Setup and Running the Application

### Prerequisites

*   Python 3.9+
*   `pip` (Python package installer)
*   Docker (for containerized deployment, optional for local dev)

### Local Development (Without Docker)

1.  **Clone the repository:**
    ```bash
    git clone <repository_url>
    cd <repository_directory>
    ```

2.  **Create and activate a virtual environment (recommended):**
    *   On macOS and Linux:
        ```bash
        python3 -m venv venv
        source venv/bin/activate
        ```
    *   On Windows:
        ```bash
        python -m venv venv
        venv\Scripts\activate
        ```

3.  **Install dependencies:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Run the application:**
    ```bash
    python app/main.py
    ```
    This command will start the Flask development server. The first time it runs, it will create `app/database.db` and the `uploads/` directory if they don't exist.

5.  **Access the application:**
    Open your web browser and go to `http://localhost:5001`.

### Running with Docker

1.  **Ensure Docker is installed and running.**

2.  **Build the Docker image:**
    From the project root directory (where the `Dockerfile` is located):
    ```bash
    docker build -t photo-album-app .
    ```

3.  **Run the Docker container:**
    ```bash
    docker run -p 5001:5001 \
           -v ./my_uploads_data:/usr/src/app/uploads \
           -v ./my_app_data:/usr/src/app/app \
           photo-album-app
    ```
    *   **Explanation of volume mounts (`-v` flags):**
        *   `-v ./my_uploads_data:/usr/src/app/uploads`: This mounts the `my_uploads_data` directory from your host machine (it will be created if it doesn't exist) to the `/usr/src/app/uploads` directory inside the container. This ensures that photos uploaded by users are persisted on your host machine even if the container is stopped or removed.
        *   `-v ./my_app_data:/usr/src/app/app`: This mounts the `my_app_data` directory from your host to `/usr/src/app/app` inside the container. This is used to persist the SQLite database (`database.db`) and any temporary zip files (`tmp_zip_files/`) generated by the application.

    *   The application will be accessible at `http://localhost:5001` (or `http://<your_docker_host_ip>:5001` if running Docker on a remote machine or VM).

## Running Tests

1.  Ensure you are in the project root directory.
2.  Make sure dependencies are installed (as per local development setup).
3.  Run the unit tests:
    ```bash
    python -m unittest discover -s tests -v
    ```
    (The `-v` flag enables verbose output.)

## Customization

*   **Themes:** The color theme of the application can be changed via links in the navigation bar of the web UI.
*   **Album Names:** Album names are set when an album is created through the UI.

## Emailing

*   **Simulation Only:** The current email sharing feature is a simulation. Instead of sending an actual email, it prints the intended recipient, subject, body, and attachment details to the console (standard output of the Flask application). To implement actual email sending, the `send_email_with_attachment` function in `app/main.py` would need to be updated with SMTP server details and credentials.
